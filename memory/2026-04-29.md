## WSL → Windows GitHub 推送解决方案

### 问题
- WSL 中 git push 到 GitHub 一直失败（connect failed to port 443）
- 代理配置不生效

### 解决方案（用户建议）
1. 将 WSL 项目文件复制到 Windows 临时目录
2. 在 Windows 中用 git 推送
3. 清理临时文件

### 操作步骤
```powershell
# 1. 复制文件到 Windows
wsl cp -r /home/administrator/projects/secure-vault-sol/* /mnt/c/temp-secure-vault/

# 2. Windows git 推送
cd C:\temp-secure-vault
git config http.proxy http://127.0.0.1:7890
git push origin master

# 3. 清理临时目录
cmd /c "rd /s /q C:\temp-secure-vault"
```

### 结果
- ✅ secure-vault-sol 项目成功推送到 https://github.com/529349029/secure-vault-sol
- ✅ 临时目录已清理

## Git 推送模式

### WSL → GitHub 推送方案
**问题**：WSL 中 git push 到 GitHub 失败（443 端口连接问题）
**方案**：复制到 Windows → 用 Windows git 推送 → 清理临时文件

```powershell
# 1. 从 WSL 复制到 Windows
wsl cp -r /path/to/project/* /mnt/c/temp-project/

# 2. Windows git 推送（配置代理）
cd C:\temp-project
git config http.proxy http://127.0.0.1:7890
git push origin master

# 3. 清理
cmd /c "rd /s /q C:\temp-project"
```

**记住**：以后 WSL 推 GitHub 有问题就这套流程
