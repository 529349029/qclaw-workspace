# 2026-04-29 工作日志

## WSL → Windows GitHub 推送解决方案

### 问题
- WSL 中 git push 到 GitHub 一直失败（connect failed to port 443）
- 代理配置不生效

### 解决方案（用户建议）
1. 将 WSL 项目文件复制到 Windows 临时目录
2. 在 Windows 中用 git 推送
3. 清理临时文件

### 操作步骤
```powershell
# 1. 复制文件到 Windows
wsl cp -r /home/administrator/projects/secure-vault-sol/* /mnt/c/temp-secure-vault/

# 2. Windows git 推送
cd C:\temp-secure-vault
git config http.proxy http://127.0.0.1:7890
git push origin master

# 3. 清理临时目录
cmd /c "rd /s /q C:\temp-secure-vault"
```

### 结果
- ✅ secure-vault-sol 项目成功推送到 https://github.com/529349029/secure-vault-sol
- ✅ 临时目录已清理

## 文件操作原则 - 用户原文件保护

### 事件
用户非常不满，因为我直接修改了 D:\usdt\git-sync.bat 导致原文件丢失

### 教训
**永远不要直接修改用户的原文件**：
- 如果需要修改用户仓库中的脚本，必须先复制一份
- 在复制份上修改测试
- 原文件保持不变
- 用 `.original` 或 `.bak` 后缀命名复制

### 恢复操作
- 用 `git show` 从 git 历史恢复原内容到 `git-sync.bat.original`
- 用户保留原文件 `git-sync.bat` 不动

### 原则写入 TOOLS.md
以后配置环境或修改脚本时：
1. 先复制到工作区
2. 在复制份上操作
3. 不动用户的原始文件
